# Schema Contract — Relational vs JSONB Split

## Architecture Overview

The DB uses a **hybrid model** (since migration 0010):
- **Relational columns** — shared fields that exist on every product regardless of category
- **JSONB `attributes`** — category-specific fields defined by the Zod schema registry

Both `products` and `productVariants` tables have an `attributes JSONB` column with a GIN index.

**NOT relational (moved to JSONB in migration 0010):**

## Zod Attribute Schema Registry

Located at `src/lib/validations/attributes/index.ts`. Maps category slugs to Zod schemas.

### Resolution chain

```
1. Exact match: "wc" → wcAttributeSchema
2. Parent walk: "imitation-marbre" → "imitation" (no match) → root
3. Root prefix: "carrelage..." → tileBaseAttributeSchema
4. No match → null (permissive, no validation)
```

### Current schemas by category

#### Tiles (`carrelage` root)

**Product-level attributes:**
| Key | Type | Values | Notes |
|-----|------|--------|-------|
| `productType` | enum | `sol`, `mur`, `sol-et-mur` | |
| `variation` | enum | `V1`, `V2`, `V3`, `V4` | V1=Uniforme, V2=Légère, V3=Modérée, V4=Forte |
| `rectified` | boolean | | |
| `fullBody` | boolean | | |
| `boxCoverage` | number | | m² per box — `variantField: true` |
| `piecesPerBox` | number (int) | | — `variantField: true` |

**Floor tiles** add: `wearResistance` (PEI-1..PEI-5), `antiSlipRating` (R9..R13)
**Mosaic** adds: `meshMounted` (boolean), `chipSize` (string)

#### Sanitary (`salle-de-bain` root)

**Base (all sanitary):**
| Key | Type | Values |
|-----|------|--------|
| `material` | enum | `ceramic`, `porcelain`, `resin`, `solid-surface` |
| `mountType` | enum | `wall-hung`, `floor-standing`, `countertop` |

**WC** adds: `evacuationType` (horizontal/vertical/universal), `seatHeight` (number mm), `rimless` (boolean)
**Bidet:** inherits base only
**Reservoir** adds: `flushVolume` (string, e.g. "3/6L")
**Sink** adds: sink-specific fields

#### Shower (`douche` root)

**Base:** `glassTreatment` (string), `glassThickness` (number mm)
**Cabin** adds: `doorType` (sliding/pivot/folding), `shape` (rectangular/quarter-circle/walk-in)

#### Furniture (`meuble-salle-de-bain` root)

**Base:** `furnitureMaterial` (MDF/solid-wood/melamine), `color` (string, `variantField: true`)
**Vanity** adds: `numberOfBasins` (number), `countertopType` (ceramic/stone/solid-surface)

## Field Placement Decision Tree

```
Is this field a column in products/productVariants table?
  YES → put in "relational": { ... }
  NO ↓

Is this field defined in the resolved Zod schema?
  YES ↓
    Is it marked variantField: true?
      YES → put in variant "attributes": { ... }
      NO  → put in product "attributes": { ... }
  NO ↓

Is it catalog-specific data not in any schema?
  YES → put in product "attributes": { ... } (JSONB is permissive)
  NO  → omit
```
